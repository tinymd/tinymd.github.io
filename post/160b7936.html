Lecture 1

发表于 2020-06-24

1、 Intro to NLP

NLP的目标：设计算法使得电脑“懂得”人类语言，便于执行更多任务。目前常见任务由易到难主要分为一下几点。

如何表示单词？这是首先必须解决的问题。早期以独立的符号表示各个词，如one hot，\(\left[\begin{array}{c}0 &1 &0 &\ldots &0\end{array}\right]\)，无法表示词之间的相似度和差异度，且这样的词向量极其稀疏，占用内存也很大。因此需要一种更加稠密的向量，且向量本身封装着单词的含义，这种向量就是后文中介绍的词向量。

2、 词向量介绍

在2013年之前，这种封装含义的稠密向量-词向量一般都通过SVD的方法得到，2013年，Mikolov提出了word2vec，word2vec是一个软件包：2种算法-CBOW和skip-gram，2种加速训练方法-负采样和层级Softmax。

2.1、CBOW和skip-gram介绍

CBOW和skipgram，这两种办法都是衡量一个句子的合理性来推出单词的词向量表示。遍历语料库，CBOW的目标是希望能够预测任一单词周围的单词，skip-gram的目标是希望能根据周围单词预测中心的单词。本章分三个部分来讲解两种办法：1）损失函数，2）如何定义概率，3）如何最优化。

2.1.1、损失函数

将CBOW和skip-gram两者目标数学化表示：

CBOW：最大化概率\(L(\theta)=\prod_{t=1}^{T}P\left(w_{t} \mid w_{t-m}, \ldots, w_{t-1}, w_{t+1}, \ldots, w_{t+m}\right)\)

skip-gram: 最大化概率\(L(\theta)=\prod_{t=1}^{T} \prod_{-m \leq j \leq m \atop j \neq 0} P\left(w_{t+j} \mid w_{t} ; \theta\right)\)

其中，t为单词所有的位置表示，T表示语料库中所有的单词数，m表示限定的窗口大小，\(w_t\)表示语料库中第t个单词，\(\theta\)为所有未知参数。通常为求导与计算方便，我们将最大化问题转换为如下的最小化问题：

CBOW: 最小化概率\(J(\theta)=-\frac{1}{T} \log L(\theta)=-\frac{1}{T} \sum_{t=1}^{T}\log P\left(w_{t} \mid w_{t-m}, \ldots, w_{t-1}, w_{t+1}, \ldots, w_{t+m}; \theta\right)\)

skip-gram: 最小化概率\(J(\theta)=-\frac{1}{T} \log L(\theta)=-\frac{1}{T} \sum_{t=1}^{T} \sum_{-m \leq j \leq m \atop j \neq 0} \log P\left(w_{t+j} \mid w_{t} ; \theta\right)\)

2.1.2、如何定义概率

此时的最重要的问题：如何计算\(P\left(w_{t} \mid w_{t-m}, \ldots, w_{t-1}, w_{t+1}, \ldots, w_{t+m}; \theta\right)\)和\(P\left(w_{t+j} \mid w_{t} ; \theta\right)\)，两者原理类似，以后者(skip-gram)为例来说明：

假设任一单词\(w\)有两个词向量\(v_w\)和\(u_w\)，分别对应\(w\)为上下文和中心词的情况。则定义：

\(P\left(w_{t+j} \mid w_{t} ; \theta\right)=P(u_{w_{t+j}} \mid v_{w_t})=\frac{\exp \left(u_{w_{t+j}}^{T} v_{w_t}\right)}{\sum_{w \in V} \exp \left(u_{w}^{T} v_{w_t}\right)}\)，简化公式表达，中心词为\(c\)，上下文词为\(o\)，则根据\(c\)预测\(o\)的概率为:

\(P(o \mid c)=\frac{\exp \left(u_{o}^{T} v_{c}\right)}{\sum_{w \in V} \exp \left(u_{w}^{T} v_{c}\right)}\)

解释此概率定义方法，通常我们求两向量的相似度通过内积实现，因此此处通过计算上下文词向量与中心词向量的内积来求得两向量的相似度，内积的范围\((-\infin,+\infin)\)，因此使用Softmax方法将此数值转换为概率。

因此在此概率模型下，我们需要确定的模型参数只有所有单词的上下文词向量和中心词向量。

2.1.3、如何最优化

迭代学习参数的经典方法-梯度下降。

推导如下：

\(\frac{\partial J(\theta)}{\partial v_{w}}=\sum_{t \in Ce}\frac{\partial}{\partial v_{w_t}} \sum_{-m \leqslant j \leqslant m} \log P(u_{w_{t+j}} \mid v_{w_t})=\sum_{t \in Ce}\sum_{-m \leqslant j \leqslant m} \frac{\partial}{\partial v_{w_{t}}} \log P(u_{w_{t+j}} \mid v_{w_t})\)

\(\frac{\partial J(\theta)}{\partial u_{w}}=\sum_{w_{t} \in T_{w}} \frac{\partial}{\partial u_{w_t}} \sum_{-\pi+j \leqslant m\atop j \neq 0} \log \left(u_{w_{t}} \mid v_{w_{t+j}}\right)\)

其中，\(T_w\)表示语料库中单词\(w\)出现的所有位置

这种方法需要学习的主要参数都是中心词向量和上下文词向量两个矩阵，每个单词有两个向量，（官方文档中w2v使用v表示上下文词向量，u表示中心词向量，而skipgram相反，不要混淆，可简洁即为v表输入，u表输出)。首先理解两种办法学习词向量的逻辑（或者说为什么这样的向量矩阵是合理的），确定损失函数是什么，再了解加速学习参数的两种办法-负采样和层级Softmax。 一、两种办法 \1. CBOW使用上下文预测中心的办法，（文档中略去初始化未讲，可以随意假设随机初始化，不影响理解算法思想），平均窗口2m内所有上下文向量作为该位置的上下文向量，再计算该上下文向量与中心向量的内积，最后使用Softmax转换为概率，我们希望的是真实单词的中心向量应取得最大值。 \2. skipgram使用中心预测上下文，一个中心词向量输入，计算上下文词向量与该中心词向量的内积，使用Softmax转换为概率，最大化2m个上下文位置上的概率乘积。个人对于skipgram的理解：当我们有很多句子的时候，一个单词周围出现的2m个单词会有很多套，这样保证了两点：第一，不会存在两单词上下文词向量相同的情况（因为单看一个样本即一个中心词时，周围不同位置的单词其上下文词向量并不会根据此损失函数而有所不同，但样本多了就不一样了），第二，出现频率最高的两个单词的上下文向量和中心词向量会很接近。
