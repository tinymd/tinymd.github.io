一、准备工作

1. Github创建个人仓库

搭建个人博客需要一个唯一的域名，我们可以自己购买，但我觉得一般使用的话用免费的就好啦，具体过程与在github上新建repository基本一致，关键在于新仓库的名字，一定是UserName+"github.io"的形式，如下图所示，忽略红字警告，这是因为我之前已经申请好了。

2. 安装Git

Git是开源的分布式版本控制系统，我们网站在本地搭建好了，需要使用Git同步到GitHub上，Git细节参考廖雪峰老师的教程。

3. 安装node.js

Hexo基于Node.js，Node.js下载地址：Download | Node.js下载安装包，安装时一直默认即可，会自动包含环境变量及npm的安装，安装后，检测Node.js是否安装成功，在命令行中输入 node -v 和npm -v。

4. 在本地安装Hexo

Hexo是高效的静态站点生成框架，通过 Hexo 你可以轻松地使用 Markdown 编写文章，除了 Markdown 本身的语法之外，还可以使用 Hexo 提供的标签插件来快速的插入特定形式的内容，官方文档由中文编写。

Hexo就是我们的个人博客网站的框架， 在自己电脑常里创建一个文件夹，可以命名为Blog，Hexo框架与以后你自己发布的网页都在这个文件夹中。创建好后，进入文件夹中，按住shift键，右击鼠标点击命令行，使用npm命令安装Hexo，输入：

npm install -g hexo-cli

安装好之后，进入之后存放博客的文件夹，输入命令：

hexo init <filefoldername>，比如hexo init testblog。

然后testblog新建成功，进入此文件夹，输入命令:

hexo g && hexo s

点开http://localhost:4000,然后会有下图中的页面，此时本地已经搭建好博客

接下来就剩下部署到GitHub Pages了~

2. 安装git部署插件

按照官网Github Pages的使用方法，获得一个网站总共分3步，

  1. 新建仓库
  2. 本地clone repository
  3. 写html文件
  4. Add, commit, and push your changes

到目前为止，我们可以在本地写html文件，然后部署到Github仓库里，但是hexo有Git部署插件，可以使得我们写博客时不用写html文件且部署步骤简单。Git部署插件，输入命令:

npm install hexo-deployer-git --save

这时，我们分别输入三条命令：

hexo d 就是部署网站命令，d是deploy的缩写。完成后，打开浏览器，在地址栏输入你的放置个人网站的仓库路径，即 http://xxxx.github.io，你就会发现博客已经上线了。

三、优化

1. 更换主题

如果不喜欢hexo默认的主题，可以更换其他的主题，主题传送门，比较流行的是next主题，我自己使用的也是next主题。在根目录中打开命令行输入：

git clone https://github.com/theme-next/hexo-theme-next themes/next

2. 优化网页样式布局

如此图中的间距，如果想要缩小或增大,具体有以下步骤：

3. hexo插件

如本文就使用了hexo-toc插件，可以在文本前插入一个目录。
